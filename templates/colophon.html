A brief history

This Web site has evolved from a simple blog which was maintained by hand, in raw HTML. In 2004, I switched to the now-defunct pMachine CMS because the manual archiving and indexing process was becoming a royal pain. pMachine was replaced by WordPress a mere three months later and I kept with it for a good four years.

I started playing with Django in late 2007 while working at The Maneater Student Newspaper. Along the way, I picked up the urge to build myself a blog — from scratch, in Django. It seems a common rite of passage among Django developers these days.

In 2008, I broke down and did it. In early 2009, the site went through a minor redesign. In 2010, the site was completely rewritten to cut out some of the messier bits.

Design

  • Pages are 500px wide, in a one-column setup.
  • Main header fonts are generally Helvetica Neue Light, or if that’s not available, Arial (or another system sans-serif) is used instead.
  • The body (and small header) font is Gentium Basic.
  • Custom fonts (including Gentium Basic) are loaded via the CSS3 @font-face tag, rather than SIFR or Cufón. You can read about that here or here.
  • And again, this is open all source. You can find the design-specific files here and here within the repository.

Backend

  • The meat-and-potatoes of the site consist of a custom-built blogging app (among other tools) programmed in Python and powered by the Django framework.
    • I’m using Django’s trunk version and update it frequently. My server says I’m running the following Django version: {% current_django_version %}.
  • Some other dependencies:
    • I use typogrify to keep headlines neat and tidy. (Which in turn uses Smartypants.)
    • django-contact-form powers the contact form.
    • akismet.py helps the contact form filter out spam submissions.

Hosting

  • This site is hosted on a 256 slice at Slicehost, in my good ol’ hometown of St. Louis, Missouri. They rock.
  • My server runs Ubuntu 9.10 using nginx in front of Apache, which serves Django via mod_wsgi. Data lives in PostgreSQL. Memcached is used here and there to speed things up a bit — nginx even directly serves pages from the Django cache, for awesome performance.
    • Some example configuration files are stored within the repository, if you”re interested in how I did this
  • I generally upload multimedia files to Amazon S3/CloudFront.
  • Text-based static files (like the site’s styles) are served on a Google App Engine instance, since Amazon S3 doesn’t gzip compress.
    • You can view my app.yaml and my “push to server”script if you want to know how I get things into my App Engine instance.
  • Oh, and I register most of my domains through GoDaddy and the tig.as domain is registered directly at the .AS Domain Registry. I manually edit my DNS records in SliceHost’s panel.
